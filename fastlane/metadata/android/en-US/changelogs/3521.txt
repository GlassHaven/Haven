v5.24.88: unified keystore with biometric protection + zh-CN i18n fixes

#129 lands end-to-end. The Keys tab now doubles as the security
audit screen: capability chips per key (Hardware-backed, Passphrase,
User presence / verification), a "Require biometric to use" toggle
on each SSH key, and a stored-passwords section showing every
encrypted profile credential with one-tap wipe.

When a biometric-protected key is fetched (SSH connect, key list
walk), Android's BiometricPrompt fires before the bytes leave the
keystore. A 30 s session-unlock window collapses repeat prompts
during a single connection that walks multiple keys. Backgrounding
the app re-locks immediately. Profile passwords still flow through
the existing per-profile auth path.

Internals: new core/security Keystore interface (enumerate, fetch,
wipe, exportAudit, setBiometricProtected) with adapters over the
ssh_keys table and ConnectionProfile password columns, DB
migration 41→42 adding biometricProtected, and the SSH auth path
(SshKeyRepository) routes through the gate. 54 unit tests pin the
contract.

#125: zh-CN translations — 67 keys whose values had been left as
English are now actually translated (Show Desktops/Linux VM card,
SFTP sync/convert dialog, terminal tab errors, etc). All ten
Settings section headers extracted + translated. Connection-edit
dialog field labels (Transport / Label / Host / etc) remain in
English; that's a separate ~30-string arc.

SSH certificate auth (#133 phase 1): keys can now have an OpenSSH
certificate (id_xxx-cert.pub) attached via the Keys tab long-press
menu. When connecting, JSch validates against the cert's CA
signature so step-ca / Authentik / Okta SSH-cert flows work after
a manual import. OIDC + auto-renewal is the next phase.

Plus: robot icon on Connections now opens the agent activity log
directly instead of dropping you in Settings.
